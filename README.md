# 商城项目手工测试记录

## 📋 项目介绍
1. 业务特性：项目是用来干啥的？
   一个个人对个人的借款平台。
2. 项目主要用户和角色？
  - 借款人：发布借款申请
  - 投资人：针对借款人发布的借款申请进行投资
  - 平台审核员：针对借款人的资质、投资记录进行相关审核
  - web前台
  - app前台
  - web后台管理系统
3. 项目核心业务及模块有哪些？
  - 借款业务：开始 → 借款人注册并登录 → 开通资金托管账号 → 申请额度及审核 → 发布借款申请及审核 → 
结束
  - 投资业务：开始 → 借款人注册并登录 → 投资 → 结束
  - 前置条件：
      - 待投资的借款单据
      - 已完成风险评测
      - 账户余额成功
4. 项目技术实现方式？
  - 特点：采用前后端分离（对前端界面进行功能测试，对后端代码进行接口测试）
  - 技术：
     - a.前端：HTML + CSS + JS + Vue
     - b.后端：Java
     - c.数据库：Mysql
     - d.web服务器：Nginx
5. 测试流程 
  - 需求分析评审
  - 编写测试计划
  - 设计测试用例
  - 执行测试用例
  - 缺陷跟踪管理
  - 编写测试报告

## 借款流程
1. 注册登录 开通资金托管
2. 前台申请额度 后台额度管理的额度列表进行审核通过
3. 前台个人主页-点击品质理财-个人借款-信用标（立即借款）
4. 后台初审管理-初审标（审核通过）
5. 前台智能投顾-投资列表（投标）

## 业务流程测试步骤
1. 注册并登录（判定）
2. 开户成功
  - 填写开户信息
  - 调用第三方开户系统
3. 额度申请
  - 提交额度申请
  - 管理员后台登录
  - 管理员进行审核通过
4. 发布借款成功
  - 提交借款申请
  - 管理员后台登录
  - 管理员审核借款申请
